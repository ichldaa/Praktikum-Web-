Studi Kasus: Praktik Konseling Care

Ringkasan Kasus

Praktik Konseling Care mencatat data pasien, layanan konseling, dan transaksi pembayaran. Sistem ini bertujuan untuk memudahkan pencatatan dan pelaporan.

Entitas Utama

  • Pasien
  • Konselor
  • Layanan
  • Transaksi

Alur Proses

  1. Pendaftaran
  2. Pencatatan Layanan
  3. Pembayaran
  4. Laporan

Contoh Tabel Data

Nama Field Tipe Keterangan
pasien_id INT Primary Key
nama_pasien VARCHAR Nama lengkap pasien
layanan_id INT Relasi ke tabel layanan
tanggal DATE Tanggal layanan

Laporan Praktikum (Laprak)

Kata Pengantar

Puji syukur ke hadirat Tuhan Yang Maha Esa karena laporan ini dapat diselesaikan. Laporan ini dibuat untuk memenuhi tugas praktikum Sistem Informasi.

BAB I - Pendahuluan

Latar Belakang: Sistem informasi membantu pencatatan data pasien, layanan, dan transaksi di Praktik Konseling Care.

Rumusan Masalah:

  • Bagaimana merancang struktur data pasien dan layanan?
  • Bagaimana membuat alur transaksi pembayaran terintegrasi?

Tujuan:

  • Membuat model data konseling.
  • Merancang tampilan input data dengan HTML & CSS.

BAB II - Tinjauan Pustaka

Sistem informasi manajemen, basis data relasional, dan normalisasi data.

BAB III - Metode

Pengumpulan data dilakukan dengan observasi dan studi literatur. Perancangan menggunakan analisis kebutuhan, normalisasi, dan pembuatan prototipe sederhana dengan HTML & CSS.

BAB IV - Hasil dan Pembahasan

Tampilan halaman HTML & CSS sederhana yang menggambarkan alur sistem konseling.

BAB V - Kesimpulan

Sistem informasi sederhana dapat membantu pengelolaan data pasien, layanan, dan transaksi.
